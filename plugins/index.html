principe
  • ListenToMe permet de pouvoir donner des ordres sans devoir répéter SARAH à chaque fois.

  • Pour cela, il crée des copies des XML principaux de vos plugins et en fait des XML lazy , c'est à dire des XML qui, par défaut, ne sont pas chargés par la grammaire du client Sarah. Il modifie ces XML en supprimant le mot SARAH dans la ou les occurences <item>Sarah

  • Lorsque que vous activez ListenToMe, les XML d'origines de vos plugins sont ignorés et seuls les XML lazy sont chargés par le client SARAH. Ainsi vous pouvez donner vos ordres sans devoir répéter Sarah puisque ce mot à été supprimé.

Utilisation et Configuration
  • Cliquer sur le lien un peu plus bas pour afficher la page de sélection.
    Avant d'afficher la page, ListenToMe liste et contrôle les XML principaux de vos plugin
    sélectionnez ceux de votre choix puis valider.
    En validant, ListenToMe va créer les XMLlazzy des plugins sélectionnés.
    ListenToMe va également modifier son propre XML pour paramétrer le mode lazy.

    Page de contrôle et de sélection des plugins

  • Maintenant, il suffit de dire à Sarah:
    - Sarah, écoute moi s'il te plait
    Sarah vous réponds:
    - Je t'écoute
    Vous pouvez maintenant commander vocalement les plugins choisis sans devoir dire Sarah


  • pour quitter immediatement ce mode, il suffit de dire à Sarah:
    - Merci ça sera tout, SARAH
    Sarah vous réponds:
    - Je t'en pris
    La grammaire normale est maintenant rechargée par Sarah.
    Le mode Lazy n'est pas permamant (Cf Info sur le lazy)


  • pour désactiver un pluggin, il suffit de retourner sur la page de configuration, de le déselectionner puis valider.


Infos et Compatibilités
  • ListenToMe ne traite que le XML principale du plugin (nom_du_plugin.xml).
    Par exemple, pour xbmc, il ne traitera que xbmc.xml mais ne touchera pas à lazyxbmx.xml ou xbmcperso.xml

  • ListenToMe crée le fichier XML avec ce nom : lazyLTMnom_du_plugin.xml


  • Pour être compatible avec ListenToMe, le XML du plugin doit contenir:
    <item>Sarah ou <item>SARAH ou <item>sarah ...
    et non :
    <item  >Sarah
    <  item>Sarah
    <item>  Sarah
    <item>Bonjour Sarah
    <item>Jarvis
    <item weight="2">Sarah
    Si un plugin ne respecte pas ce format, il suffit de corriger le XML d'origine.
    Par exemple, pour les plugins telovh, homeseer, dateconviviale (...) il suffit de retirer un espace de trop ou un weight="2" inutile.

  • Les XML crées sont stocké dans le dossier du plugin ListenToMe.

  • Les XML Lazy des pluggins désactivés ne sont jamais supprimés, il sont seulement ignorés.

  • Si un pluggin est mise à jour sur Sarah, il suffit de retourner sur la page de configuration de ListenToMe et de valider.
    Le XML lazy sera recrée.

Info sur le lazy
    Le lazy (ou context de grammaire) de SARAH n'est pas permanant, après 60 secondes d'inactivité, il s'arrête .
    Il est possible de configurer ce temps (en milliseconde) dans le fichier custom.ini à la racine de Sarah:
          ; Reset grammar to default after given timeout (millis)
          ctxTimeout=60000
